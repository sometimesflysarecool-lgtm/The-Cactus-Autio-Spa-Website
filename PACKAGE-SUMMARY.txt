PROBLEM: CSS didn't take effect
SOLUTION:
  1. Clear browser cache (Ctrl+Shift+Del)
  2. Verify entire CSS was copied (check last line in Squarespace)
  3. Click "Save" in Custom CSS panel
  4. Wait 30 seconds for Squarespace to deploy
  5. Test in incognito window (no cache)

PROBLEM: Still seeing horizontal scrollbar
SOLUTION:
  1. Check browser console for CSS errors (F12 → Console)
  2. Verify no conflicting !important rules elsewhere
  3. Test on a different browser
  4. Test on actual mobile device (not just DevTools emulation)
  5. Check for custom Squarespace plugins blocking the CSS

PROBLEM: Desktop layout changed
SOLUTION:
  1. Media queries should only affect (max-width: 768px)
  2. Desktop should be completely unchanged
  3. If broken: clear CSS and try QUICK-REFERENCE-CSS.txt version
  4. Contact support if issue persists

PROBLEM: Some elements still overflow
SOLUTION:
  1. Inspect element with DevTools (right-click → Inspect)
  2. Check computed width vs viewport width
  3. Look for inline styles overriding the CSS
  4. Try adding specific selector to the CSS for that element

Q: Will this affect my desktop layout?
A: No. All fixes are inside @media (max-width: 768px) which only applies to
   mobile/tablet. Desktop (769px+) is completely unchanged.

Q: Do I need to modify my HTML?
A: No. This is CSS-only. No HTML or JavaScript changes needed.

Q: How long does it take to implement?
A: 2 minutes to apply. Testing takes 5-10 minutes depending on device count.

Q: Will this break anything?
A: Low risk. These are standard CSS best practices. If issues occur, the CSS
   can be removed from Squarespace Custom CSS in seconds.

Q: Can I edit the CSS after applying?
A: Yes, you can customize padding/margin values to match your design
   preferences, but the core overflow-x: hidden and box-sizing rules should
   remain unchanged.

Q: Does this work on all browsers?
A: Yes. These CSS properties are supported in all modern browsers
   (Chrome, Firefox, Safari, Edge) and IE11+.

Q: What if I still see scrolling issues?
A: See the Troubleshooting section. Most common issue is browser cache.
   Clear cache and try again.
